# Use Cases

What is this tool?
This is an online service for turning children drawings, scribbles, and paintings into the custom 3D-printed toys.

use-case:
Register/Login:
User can register and create an account by using their email and choosing a password or alternatively they can use google authenticate to use the service. They should be able to recover their password if they forgot it.

Upload image file:
User in the Hero section should see 'A platform to make your drawings into real toys' title size and a smaller size description 'Transform any drawing, scribble, doodle and mark-making into real toys and preview it for free' and the drag and drop area to upload their image. The button should says ' Get started for free'

Preview the generated concept and original image:

1. As soon as the image uploaded, user should see the next screen which is the preview.
2. User should be able to rename their toy, and see the image they uploaded as 'Original image' next to the 'Generated concept' which is made by an image-text-to-image AI model. User can only choose one to continue.
3. User should be able to view and edit the generated 'Toy's story' which is a short description of the image in a story telling wey.
4. User should be able to Choose either 'Fully Crafted Toy' and/or 'DIY Toy' options. These options are cards with the following details:
   4.1. Fully Crafted Toy: £39, VAT included, Fully crafted & colored, Ready in 3-5 working days, No coloring set, PLA material(Eco-friendly), Free shipping, Up to 25 cm size range.
   4.2. DIY Toy: £29, VAT included, Uncolored, Ready in 2-3 working days, Includes coloring set, PLA Material (Eco-friendly), Free shipping, Up to 30 cm size range.
5. User need to check mark the consent sentence ' I am happy with the selected name, story, and style to continue' to continue.
6. User should be able to go back(Back) to upload a new drawing or continue to the next screen (Add to cart ->)

Shopping cart:

1. User should be able to choose and delete any item in the cart in a table with the order name, reference image (file name), price, type (DIY toy or Fully crafted toy), Quantity. User should also change the number of quantity.
2. User on the right side of the screen should see the 'Cart total' which includes information like subtotal, shipping, editable discount code input with 'Apply' button, applied discount, and the total price.
3. User should check mark "I've read the terms of service and happy to proceed with the payment" before continue to the next screen.
4. User should be able to go back to the previous screen.
5. User should be able to come back to the 'Shopping Cart' by the icon button in the nav bar.

Billing Details:

1. User should enter and edit their billing address with the following inputs: Full name, Email, Phone number, Postcode and the 'Find address' button to autofill the address, Address line 1, Address line 2, and delivery instruction.
2. The 'Cart total' on the right side should be same as previous step.
3. User should be able to go back to the previous page(Shopping Cart) or continue to the next screen which is 'Checkout'.

Payment screen is external service 'Stripe'

About:
User should see the about page.

User should see the Privacy page.

User should see the Terms of service page.
